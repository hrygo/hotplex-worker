# 移动端访问

> 通过飞书/Slack 移动应用随时随地与 AI 对话

## 概述

HotPlex Gateway 支持通过飞书（Feishu）和 Slack 的移动应用与 AI Agent 对话。这意味着你可以在通勤、会议间隙或任何不方便使用电脑的场景下，直接通过手机与 AI 交互。

## 飞书移动端

### 前提条件

- 飞书机器人已配置并连接到 HotPlex Gateway
- 你拥有飞书机器人的访问权限

### 使用方式

1. 打开飞书移动应用
2. 找到已配置的 AI Bot
3. 直接发送消息即可开始对话

### 功能支持

- 文字消息交互
- 语音消息（录音 → STT → AI 处理 → TTS 语音回复）
- 富文本卡片消息（代码块、表格、列表）
- 斜杠命令（`/gc`、`/reset`、`/compact` 等）

## Slack 移动端

### 前提条件

- Slack Workspace 已安装 HotPlex Bot 应用
- 你拥有 Bot 的使用权限

### 使用方式

1. 打开 Slack 移动应用
2. 在 DM 或已添加 Bot 的 Channel 中发送消息

### 功能支持

- 文字消息交互
- 语音消息（录音 → STT → AI 处理 → TTS 语音回复）
- 代码块格式化显示
- 斜杠命令支持

## 语音交互

HotPlex 支持完整的语音交互循环：

```
录音 → 语音转文字（STT）→ AI 处理 → 文字转语音（TTS）→ 音频回复
```

### 发送语音消息

1. 在飞书或 Slack 中按住录音按钮
2. 说出你的问题或指令
3. 松开发送

AI 会自动将语音转为文字进行处理，回复时可以选择语音消息格式。

### 语音交互提示

- **清晰发音**：STT 识别准确率与发音清晰度直接相关
- **安静环境**：嘈杂背景会降低识别准确率
- **自然语速**：不需要刻意放慢速度，正常语速即可
- **简短指令**：语音消息适合简短问题，复杂任务建议用文字

## 通用命令

移动端支持与桌面端相同的斜杠命令：

| 命令 | 功能 | 移动端场景 |
|------|------|-----------|
| `/gc` | 归档会话，释放资源 | 结束对话时使用 |
| `/reset` | 重置会话，全新开始 | 话题完全切换时 |
| `/compact` | 压缩上下文 | 长对话中间 |
| `/context` | 查看 token 用量 | 检查对话长度 |

## 移动端使用技巧

### 适合移动端的场景

- **快速问答**：概念解释、语法查询、简短建议
- **代码审查**：查看 AI 分析结果、批准/拒绝操作
- **进度跟踪**：检查任务执行状态
- **灵感记录**：快速记录想法，后续在桌面端深入展开

### 不适合移动端的场景

- **复杂代码编写**：需要频繁查看多文件，适合桌面端
- **大规模重构**：涉及大量文件变更，需要完整的编辑器环境
- **长时间调试**：需要终端输出和日志查看

### 最佳实践

1. **短消息优先**：移动端输入效率低，尽量用简洁的指令
2. **善用 /gc**：对话结束后执行 `/gc`，释放服务端资源
3. **语音替代打字**：较长的问题用语音输入更高效
4. **桌面端接力**：移动端开始的任务可以在桌面端通过 Resume 继续

## 自然语言控制

除了斜杠命令，还可以通过自然语言触发控制操作。使用 `$` 前缀：

- `$休眠` 或 `$gc` —— 归档当前会话
- `$重置` 或 `$reset` —— 重置会话
- `$压缩` 或 `$compact` —— 压缩对话上下文
- `$上下文` 或 `$context` —— 查看 token 用量
- `$提交` 或 `$commit` —— 创建 Git 提交
- `$回退` 或 `$rewind` —— 撤销上一轮对话
- `$清空` 或 `$clear` —— 清空当前对话
- `$技能` 或 `$skills` —— 查看已加载 Skills
- `$切换模型` 或 `$model` —— 切换 AI 模型
- `$权限模式` 或 `$perm` —— 设置权限模式
- `$cd` 或 `$切换目录` —— 切换工作目录

完整命令参考见 [命令速查表](commands-cheatsheet.md)。

`$` 前缀确保只有明确的意图才触发控制操作，避免日常对话中的误匹配。
