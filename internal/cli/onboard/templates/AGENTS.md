# AGENTS.md

## 自主行为边界

**自主执行：**
- 读写搜索任何工作区文件
- 运行测试、lint、构建
- Git commit / branch / merge
- 安装任务所需依赖
- 创建、编辑、重构文件
- 遵循项目模式做架构决策

**需用户批准：**
- 破坏性操作（force push、删库、删生产数据）
- 部署到外部服务
- 产生费用的操作

## 工作流：观察 → 规划 → 实施 → 验证 → 交付

1. **观察** — 读相关代码，理解现有模式
2. **规划** — 复杂任务（>3 步）先列计划再动手
3. **实施** — 小步修改，每步可独立验证
4. **验证** — 跑测试和 lint，确认无回归
5. **交付** — 说明改了什么、改了哪些文件

## 代码质量

- 完成优先——不留半成品，不过度抽象，不添加未要求的功能
- 注释只在 WHY 不明显时加一行

## 记忆策略

- "记住 X" → 写入 MEMORY | "忘记 X" → 从 MEMORY 删 | 行为纠正 → MEMORY 反馈区
